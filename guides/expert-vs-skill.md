# 专家和 Skill 到底什么关系（草稿骨架）

> 旗舰文。方法论的署名是抄不走的护城河——这篇要把"全网没讲清的那件事"讲清。
> 下面是骨架，填入你的实测案例与观点后发布。

## 一句话结论（放最前）

Skill 是"能干什么"，专家是"什么时候、按什么顺序去干"。只装 skill 不给专家，等于给了工具箱没给师傅。

## 1. 大多数人的误区

- 把 Agent 能力等同于"装了多少 skill"——于是疯狂囤技能，用起来还是一团乱。
- 真正的差距不在工具数量，在"调度工具的判断力"。

## 2. 专家 = 判断力的编码

- 角色定义：身份 + 经验边界。
- 思维框架：遇到任务先想什么、后想什么（专家的灵魂）。
- 工作流程：在哪一步调用哪个 skill（把判断力落到可执行的调度上）。

## 3. Skill = 单一能力的封装

- 一个 skill 只干一件事，触发词具体。
- 好的 skill 是"专家伸手就能拿的工具"，不是"又一个需要你自己想怎么用的东西"。

## 4. 组合包 = 专家 × Skill 的化学反应

- 关键在 `pairs_with` 和「工作流程」：专家显式点名调用配套 skill。
- 举一个前后对比的实测例子：<裸装 3 个 skill 的乱 vs 组合包一次跑通>。

## 5. 为什么这是个新品类

- awesome 榜单只列 skill；本项目打包"角色 + 技能 + 实测兼容"。
- 定义权归开创者。

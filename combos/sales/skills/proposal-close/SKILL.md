# 目的
把挖到的痛点变成打动人的方案,处理异议,一步步推到成交。

# 何时使用
- 要写方案/报价,或处理异议、跟进推进成交。
- 专家在工作流程第 3 步调用。

# 输入
- 需求挖掘结果(客户痛点、影响、决策信息)
- 你的产品/报价/案例

# 步骤
1. 方案对痛点:开头复述客户的痛和影响,再给对应的解决方案(范围/怎么落地/时间线/报价),不堆功能。
2. 价值锚定:用客户自己说的代价 vs 方案投入,算 ROI。
3. 异议处理(AECR,见 [references/deal-tactics.md](references/deal-tactics.md)):承认 → 探究 → 回应 → 确认。
4. 跟进:每次接触推进一个明确 next step,别让机会晾着。
5. 逼单:给推进的理由(时间/名额/阶梯价),但不虚假紧迫。

# 输出
- 方案大纲(痛点复述 → 方案 → 落地 → 时间线 → 报价 → ROI)
- 常见异议 + 应对话术
- 跟进节奏与话术
- 本次要拿到的承诺(next step)

# 边界
- 不夸大效果、不虚假案例/比价、不制造虚假紧迫。
- 报价、承诺要能兑现;合同/价格审批走公司流程。

# 目的
用提问让客户自己说出痛点和它的代价,而不是你硬推产品。挖得越深,成交越顺。

# 何时使用
- 约到了客户,要做需求沟通/discovery call。
- 专家在工作流程第 2 步调用。

# 输入
- 客户背景、你的解决方案能解决的问题
- (可选)通话时长、对方角色

# 步骤
1. 设计 SPIN 提问(见 [references/spin-guide.md](references/spin-guide.md)):
   - S 现状:了解现在怎么做。
   - P 问题:哪里不顺、有什么困难。
   - I 影响:这个问题带来什么代价(时间/钱/风险)。
   - N 价值:如果解决了会怎样(让客户自己说好处)。
2. 排通话结构:开场 → 现状/痛点(占大头)→ 影响量化 → 价值 → 下一步。
3. 量化痛点:把"有点麻烦"逼成"每月损失多少/多花多少时间"。
4. 收口:约定明确的下一步(demo/方案/再沟通)。

# 输出
- SPIN 提问清单(按 S/P/I/N 分组)
- 通话结构与时间分配
- 需要量化的关键痛点
- 收口的下一步选项

# 边界
- 提问引导但不诱导编造;客户没说的不替他假设。
- 挖需求是为解决问题,不是套话术操纵。

# 目的
把"简历上的点"变成"面试能讲清、被追问接得住"的答案。

# 何时使用
- 已定制好简历、进面试前。
- 专家在工作流程第 3 步调用。

# 输入
- 目标岗位 JD / jd-decode 结果
- 定制后的简历(或核心经历)
- (可选)面试轮次/形式(HR/技术/终面)

# 步骤
1. 出高频题:按岗位和资历分类——通用行为题、岗位专业题、简历深挖题(针对每段经历最可能被问的点)。
2. 搭 STAR 框架:情境-任务-行动-结果;Situation 压 2-3 句、Action 占一半、结尾量化 Result,每题 1-2 分钟口语化。题库与要点见 [references/star-questions.md](references/star-questions.md)。
3. 预演追问:每个主答案配 2-3 个可能的追问及应对方向(面试官最爱往哪挖)。
4. 备缺口应对:jd-decode 标出的缺口,准备诚实且不减分的回答(怎么用相邻经历/学习计划圆场)。
5. 反问清单:结尾给几个能问面试官的好问题(显示思考、判断岗位合不合适)。

# 输出
- 分类高频题清单
- 核心经历的 STAR 答案框架(可直接改成自己的话)
- 每题 2-3 个追问 + 应对方向
- 缺口应对话术
- 面试结尾反问清单

# 边界
- 答案基于真实经历,不编造经历或数据。
- 不保证结果、不押题;给的是框架和方向,不是背诵稿。

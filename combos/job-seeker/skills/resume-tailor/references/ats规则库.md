# ATS 规则库(2026)

> 用于 resume-tailor。2026 的 ATS 已转向语义匹配,堆砌关键词反而被扣分。

## 关键词策略(2026)
- 主关键词在简历里出现 2-3 次,且分布在不同语境(概述 / 经历 / 技能各一次),别同处堆砌。
- 次关键词至少出现 1 次。
- 从 JD 原文取词,自然融入;生硬堆砌会被机审判为可疑。
- 软技能嵌进成果示例里,别单独罗列("推动跨部门上线" > 罗列"沟通能力强")。

## 板式规范(可解析优先)
- 单栏、标准小标题(工作经历 / 教育 / 技能)、日期格式一致。
- 不用表格、分栏、图形、图片、文本框。
- 存为 .docx 或可解析的 PDF。

## 成果句公式
动作动词 + 对象 + 范围/规模 + 量化结果 + 业务影响。
动作动词库:主导、搭建、优化、推动、落地、复盘、提升、降低、拆解、协调……
缺数字:留占位问句问用户,不编。

## 别做
关键词堆砌、同义词灌水、一次接触吹成熟练、花哨排版。

name: jd-decode
description: 拆解招聘 JD,分出必须项、加分项、隐藏评估点与 ATS 关键词,让求职者看清岗位真正要什么。改简历、准备面试前调用。
---

# 目的
把一段 JD 从"一堆要求"拆成"招聘方真正在筛什么"。只做拆解,不改简历。

# 何时使用
- 拿到目标岗位 JD,准备定制简历或面试前。
- 专家在工作流程第 1 步调用。

# 输入
- 目标岗位 JD 全文
- (可选)公司/团队/行业背景

# 步骤
1. 分层拆要求:
   - 硬性必须项(不满足直接被刷:学历/年限/核心技能/证书)
   - 加分项(有更好,没有不致命)
   - 噪音关键词(模板套话,别为它过度优化)
2. 提炼 ATS 关键词:技能、工具、方法论、领域术语的确切写法(简历里要原样命中)。
3. 读隐藏评估点:JD 措辞暴露的真实期待(常见措辞→考察点见 [references/jd-signals.md](references/jd-signals.md))。
4. 判断资历档位:初级/资深/管理,决定简历该突出什么(执行 vs 架构 vs 带团队)。
5. 给"匹配自查清单":逐条必须项 → 求职者需要拿什么证据对上。

# 输出
- 三分表:必须项 / 加分项 / 噪音词
- ATS 关键词清单(确切写法)
- 隐藏评估点 2-4 条
- 资历档位判断
- 匹配自查清单(必须项 × 需要的证据)

# 边界
- 只拆 JD,不改简历(那是 resume-tailor 的活)。
- 不臆测 JD 没写的要求;拿不准的标"存疑"。

# JD 措辞信号库

> 用于 jd-decode 读懂 JD 字面背后真正考察什么。

## 常见措辞 → 真实考察点
- "能独立推进 / 自驱" → 主动性、少人带也能出活
- "跨部门协作 / 沟通" → 沟通与对齐能力,别只写技术
- "抗压 / 快节奏" → 加班文化预警 + 情绪稳定性
- "从 0 到 1" → 缺流程,要能搭框架的人
- "优化 / 迭代现有" → 有成熟系统,要能接盘改进的人
- "结果导向 / 数据驱动" → 面试必问量化成果
- "Owner 意识 / 主人翁" → 责任心、端到端负责
- "学习能力强" → 岗位技术栈会变,别只会一招
- "加分项 / 优先" → 加分不是必须,别因缺它放弃投递

## 必须项 vs 噪音
- 硬门槛(学历 / 年限 / 核心证书 / 核心技能)不满足直接被刷。
- 模板套话("良好的团队精神")是噪音,别为它堆关键词。

## 资历档位信号
初级看执行、资深看架构与影响范围、管理看带人与拿结果。JD 的动词暴露档位("执行 / 负责 / 主导 / 搭建")。

# 目的
把一个研究问题变成一套能查全查准的检索方案和一份核心文献清单。只负责找,不负责读(那是 paper-digest)。

# 何时使用
- 刚定题/开题,需要摸清这个问题有哪些文献。
- 综述/论文写作前的文献收集阶段。
- 专家在工作流程第 1 步调用。

# 输入
- 研究问题/主题(尽量具体)
- (可选)学科、语言(中/英)、时间范围、可用数据库

# 步骤
1. 收窄问题:太宽就先拆成可检索的子问题(PICO 或"对象+干预+结果"框架)。
2. 建关键词矩阵:核心概念 × 同义词/近义词/英文对应(检索语法见 [references/search-tips.md](references/search-tips.md))。
3. 组检索式:布尔逻辑(AND/OR/NOT)+ 通配符,先查准再放宽查全。
4. 选数据库:按学科选(中文:知网/万方/维普;英文:Web of Science/Scopus/PubMed/Google Scholar)。
5. 筛选:按标题/摘要初筛,记录纳入/排除标准,标出高被引与近 2 年新文献。

# 输出
- 检索式(每个数据库一版)
- 核心文献清单(题录 + 一句相关性说明 + 高被引/新文献标记)
- 纳入/排除标准
- 检索日志(数据库、时间、命中数)便于复现

# 边界
- 不编造文献/DOI;查不到就说没查到,不伪造题录。
- 只找不读,精读交给 paper-digest。

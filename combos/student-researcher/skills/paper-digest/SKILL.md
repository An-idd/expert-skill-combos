# 目的
把一篇文献读透,产出结构化摘要卡,便于后续综述归纳。一次只读一篇。

# 何时使用
- 拿到核心文献,要逐篇消化。
- 专家在工作流程第 2 步调用。

# 输入
- 单篇文献(全文,或摘要 + 关键段落)

# 步骤
1. 抓四要素:研究问题、方法(设计/样本/数据)、主要结论、作者承认的局限。
2. 批判性评估:方法撑不撑得起结论?样本够不够?有没有明显偏差(评估维度见 [references/appraisal.md](references/appraisal.md))。
3. 定位贡献:它相对已有研究新在哪、补了什么缺口。
4. 记可引用点:哪些数据/结论你可能会引,标页码/出处。

# 输出
一张摘要卡:
- 题录(作者/年份/出处)
- 研究问题 | 方法 | 主要结论 | 局限
- 可信度评估(高/中/低 + 一句理由)
- 与你研究的关系(支持/反驳/方法借鉴/缺口)
- 可引用点(带出处)

# 边界
- 不曲解原文结论、不夸大;拿不准的地方标"需回原文核对"。
- 只读不写综述(归纳交给 review-outline)。

# 目的
把一堆精读摘要卡归纳成一个有逻辑的综述提纲,并点出研究缺口(缺口 = 你的选题机会)。

# 何时使用
- 已有若干篇 paper-digest 摘要卡,要写文献综述或开题报告。
- 专家在工作流程第 3 步调用。

# 输入
- 多篇文献的摘要卡(来自 paper-digest)
- (可选)综述目的/字数/学科

# 步骤
1. 归类:按主题/方法/时间/理论流派把文献分组(组织方式见 [references/review-types.md](references/review-types.md))。
2. 找脉络:每组内梳理演进——早期怎么做、后来怎么改、当前共识是什么。
3. 摆分歧:哪些问题上文献结论不一致、为什么。
4. 挖缺口:哪些问题没人做/做得不够/方法有局限——这就是你选题的落点。
5. 搭提纲:引言 → 分主题综述 → 分歧与争议 → 研究缺口 →(你的研究定位)。

# 输出
- 综述提纲(分级标题 + 每节要点 + 关键文献归属)
- 共识清单 / 分歧清单
- 研究缺口 2-4 条(每条:现状 → 缺什么 → 为什么值得做)

# 边界
- 只归纳已有摘要卡,不臆造文献没说的观点。
- 出提纲和框架,不代写可提交的综述正文。

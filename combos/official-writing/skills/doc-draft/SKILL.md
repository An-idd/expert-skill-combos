# 目的
把提纲 + 素材变成一份结构完整、语体规范的公文初稿。

# 何时使用
- 已有 doc-framework 的提纲和素材,要写正文。
- 专家在工作流程第 2 步调用。

# 输入
- 提纲(来自 doc-framework)
- 素材(往年材料、政策、数据,带出处)
- (可选)字数、行文单位、语气要求

# 步骤
1. 逐段填:按提纲每部分,把素材组织成段,段旨句在前。
2. 讲公文话:准确、庄重、简练;用规范表述("为……,现将……通知如下")替口语。
3. 用好素材:数据/政策原文照引并留出处,不编不改数字。
4. 过渡衔接:段与段之间逻辑连贯(过渡句见 [references/style-phrases.md](references/style-phrases.md))。
5. 收口:按文种写规范结尾(执行要求/下一步/号召)。

# 输出
- 公文初稿(标题 + 规范层次 + 正文)
- 用到的素材出处清单
- 存疑处标注(需核实的数据/说法)

# 边界
- 不编造数据、政策、引文;缺的标"待补/待核",不硬凑。
- 只成稿,不改框架方向(要改回 doc-framework)。

# 常用公文文种框架库

> 用于 doc-framework。以《党政机关公文处理工作条例》15 种法定文种为准,事务性文书另计。

## 选文种(先问三件事)
- 行文方向:上行(请示/报告)、下行(通知/决定/意见/批复)、平行(函)。
- 目的:告知部署(通知)、汇报(报告)、请求批准(请示)、答复(批复/函)、记录(纪要)。
- 关系:有无隶属、是不是同级。

## 常见文种骨架
- 通知:开头(缘由/依据)→ 事项(分条:做什么/谁做/时限/要求)→ 结尾(执行要求)。
- 报告:开头(基本情况)→ 主体(做法/成效/问题)→ 结尾(下一步/建议);报告不夹带请示事项。
- 请示:一文一事;缘由 → 请示事项 → 请予批复。
- 工作总结:总体概述 → 主要做法与成效(分块)→ 存在问题 → 下一步打算。
- 讲话稿:开场(定调)→ 主体(讲几点,每点 观点+论据+要求)→ 结尾(号召/希望)。
- 工作方案:背景目标 → 任务分工 → 步骤时限 → 保障措施。

## 谋篇口诀
从大到小谋篇(先定几大块),从小到大行文(再逐段填)。层次用"一、(一)、1."规范标注。

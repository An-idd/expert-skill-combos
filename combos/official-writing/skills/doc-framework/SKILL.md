# 目的
把一个模糊的写作任务变成"文种明确、框架清晰"的提纲。只搭骨架,不写正文。

# 何时使用
- 接到材料任务,还没想清楚写成什么、怎么谋篇。
- 专家在工作流程第 1 步调用。

# 输入
- 任务背景(为什么写、给谁、什么场合)
- (可选)领导的具体要求、参考的往年材料

# 步骤
1. 领会意图:问清/推断 目的、受众、场合、领导倾向;不清楚的先列出要确认的点。
2. 选文种:按行文目的和机关关系定文种(文种骨架见 [references/wenzhong-library.md](references/wenzhong-library.md))。
3. 搭框架:按该文种套路,从大到小谋篇,出分级提纲(每部分写什么、大概几段)。
4. 标素材位:每个部分标注需要什么素材/数据(便于下一步找料)。

# 输出
- 文种判断 + 理由
- 分级提纲(一、(一)、1. 到段旨)
- 每部分的素材需求清单
- 待向领导/交办人确认的点(如有)

# 边界
- 只定文种和框架,不写正文(那是 doc-draft)。
- 文种拿不准时给候选 + 各自适用条件,让用户定。

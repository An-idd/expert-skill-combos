# 目的
把"一次性成交"变成"可反复触达的会员资产":引流到私域、分层、提复购。

# 何时使用
- 大促/日常成交后要承接留存,或搭建会员/复购体系。
- 专家在"留存复购"环节调用(常在 campaign-plan 之后承接)。

# 输入
- 客户来源与规模、现有触点(微信/企微/社群/会员系统)、复购目标

# 步骤
1. 引流承接:用包裹卡/短信/客服话术把公域客户导到私域(合规,不违反平台导流规则,方式见 [references/retention-playbook.md](references/retention-playbook.md))。
2. 会员分层:按 RFM(最近购买/频次/金额)分层,新客/活跃/沉睡区别对待。
3. 复购触达:不同层给不同动作(新客首复购券、活跃上新推荐、沉睡召回)。
4. 社群/内容:社群运营节奏 + 有价值内容,别只发广告。
5. 度量:复购率、会员贡献占比、私域进人数,持续迭代。

# 输出
- 引流承接方案(触点 + 话术)
- 会员分层与权益设计
- 复购触达 SOP(分层 × 动作 × 频次)
- 度量指标

# 边界
- 遵守平台导流规则和个人信息保护(用户授权、可退订),不骚扰。
- 私域是长期资产,不做一锤子收割。

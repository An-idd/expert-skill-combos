# 目的
把一个商品的 listing(标题/主图/详情/关键词)按目标平台优化到能被搜到、点得动、转化好。一次只针对一个平台。

# 何时使用
- 上新商品,或某商品点击/转化低要优化。
- 专家在工作流程第 1 步调用。

# 输入
- 商品信息(品类/卖点/价格/受众)
- 目标平台(淘宝/天猫/拼多多/京东/抖店)
- (可选)竞品链接、现有数据(点击率/转化率)

# 步骤
1. 认平台:不同平台算法/人群/规则不同,先定这版给哪个平台(平台差异见 [references/platform-notes.md](references/platform-notes.md))。
2. 标题:核心词 + 属性词 + 长尾词组合,卡满字符、命中搜索词,不堆砌违规词。
3. 主图:第一张突出卖点/场景,符合平台尺寸;设计 3-5 张主图的信息递进。
4. 详情页:痛点 → 卖点 → 信任(资质/评价/参数)→ 促单 结构。
5. 关键词:主词/属性词/长尾词分层,匹配平台搜索习惯,埋进标题和详情。

# 输出
- 优化后标题(多版)
- 主图脚本(每张图表达什么)
- 详情页结构大纲
- 关键词清单(分层)
- 一句:这版针对哪个平台、预期改善什么指标

# 边界
- 不写违规/极限宣称,不虚假宣传。
- 一版只对一个平台,别跨平台复制。

# 目的
把"不知道发什么"变成一份可直接开写的选题清单。一次只解决选题,不写正文。

# 何时使用
- 内容日历空了、本周不知道发什么。
- 想蹭最近的热点/节日/季节,但不确定哪个跟账号相关。
- 专家在工作流程第 1 步调用。

# 输入
- 账号赛道/定位(如:平价护肤、通勤穿搭)
- 目标人群(年龄/性别/城市/消费力)
- 近期已发方向(避免重复)

# 步骤
1. 拉近 7-14 天该赛道的热点:平台热搜/热门话题、同赛道高赞新笔记的共性主题。
2. 标出未来 2-4 周的季节/节日/营销节点(换季、大促、节气、开学季等)。
3. 对每个候选选题打三个分(各 1-5 分):与账号相关度、当前热度、竞争饱和度(越低越好)。
4. 综合排序,砍掉相关度 <3 的,保留 10-20 条。
5. 每条补一句"角度"(切口类型与热点来源见 [references/topic-angles.md](references/topic-angles.md))。

# 输出
一张表,每行:
| 选题 | 内容档位(生活方式/蹭趋势/产品) | 蹭的热点/节点 | 角度一句话 | 相关度 | 热度 | 饱和度 | 优先级 |

结尾给出本周 Top 3 优先做的选题及理由。

# 边界
- 不编造热搜数据;拿不到实时热榜时,明确说明依据(如"基于近期高赞笔记归纳")。
- 只出选题,不写标题正文(那是 script-writer 的活)。

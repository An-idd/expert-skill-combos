# 目的
发布前把"必死的雷"排掉:极限词、导流、AI 造假等会直接限流/封号的问题。只做风控,不改选题方向。

# 何时使用
- 一篇小红书文案定稿后、发布前。
- 也可用于检查任何已有草稿(不限于本组合包生成的)。
- 专家在工作流程"发布前"这一步调用。

# 输入
- 待发布的标题 + 正文 + 标签(+ 可选封面文案)
- (可选)账号定位

# 步骤(先定位 → 后检测 → 再优化)
1. 定位:先明确账号赛道和目标,判断"营销浓度"是否过高(70/20/10 里超没超)。
2. 检测四类高危(清单见 [references/compliance-2026.md](references/compliance-2026.md)):
   - 极限词/绝对化用语(最、第一、绝对、100%、根治、永不反弹…)
   - 2026 新禁夸张词(绝了、封神、天花板、必入、王炸…)
   - 引流词(私信我、加微信、主页链接…)
   - 模糊夸大副词滥用(超、很、非常 堆砌)
3. 查 AI 造假风险:像不像真人真实体验,有没有 AI 腔、虚构使用感、搬运痕迹(2026 平台硬红线:回归真实)。
4. 逐条给替代写法(不只标红,要给能发的改法)。

# 输出
- 风险清单:命中词 + 类别 + 为什么会限流
- 营销浓度判断(偏高/正常)+ AI 造假风险判断
- 逐条修改建议(命中 → 建议替换)
- 一句结论:可直接发 / 改完再发 / 高风险重写

# 边界
- 只做合规参考,不替代平台实际审核;规则会变,以平台最新规则为准。
- 不做选题/文案重写(那是 script-writer 的活),只给合规改法。
- 参考清单是经验归纳,不保证覆盖所有机审词。

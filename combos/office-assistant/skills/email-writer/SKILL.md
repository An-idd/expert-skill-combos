# 目的
把一件事写成一封读得快、好回复的职场邮件。收件人几秒就能抓到重点和该做什么。

# 何时使用
- 要写请求/汇报/跟进/邀约/拒绝等职场邮件。
- 专家在工作流程第 3 步调用。

# 输入
- 邮件目的、收件人(上级/同事/客户/供应商)、关键信息
- (可选)语气要求、场景

# 步骤
1. 定场景:请求/汇报/跟进/邀约/拒绝……不同场景结构不同(话术见 [references/email-scenarios.md](references/email-scenarios.md))。
2. 写主题:直接反映内容 + 期望(如"【需确认】X 方案,请周三前回复")。
3. 开头一句:说清"为什么发这封、要你做什么"。
4. 正文:分点、简洁,重要信息前置。
5. 结尾:明确 next step(谁、什么时间、做什么)。

# 输出
- 主题行
- 正文(开头目的 + 分点内容 + 明确行动)
- (可选)不同语气版本

# 边界
- 语气得体、对事不对人;敏感/拒绝类邮件既清楚又留余地。
- 不编造事实或承诺。

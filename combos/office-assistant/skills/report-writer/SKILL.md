# 目的
把一堆做过的事,变成一份让上级一眼看到价值的汇报。汇报和升职加薪挂钩,重点是"成果"不是"忙碌"。

# 何时使用
- 要写日报/周报/月报/项目汇报/述职。
- 专家在工作流程第 2 步调用。

# 输入
- 本期做的事(流水/要点)
- (可选)汇报对象、周期、目标/KPI

# 步骤
1. 结论先行:开头一句本期最重要的成果/进展(金字塔,见 [references/report-structures.md](references/report-structures.md))。
2. 分块讲成效:按项目/模块,每块"做了什么 → 结果(量化)→ 价值"。
3. 摆问题与风险:诚实说卡点,并给已在做的对策。
4. 下一步:下期计划,和目标对齐。
5. 去流水账:合并琐事,突出重点,数字说话。

# 输出
- 一句话总结(结论先行)
- 本期成效(分块,带量化)
- 问题与对策
- 下一步计划

# 边界
- 不编造成果或数字;缺数据标"待补"。
- 突出价值但不虚报,经得起追问。

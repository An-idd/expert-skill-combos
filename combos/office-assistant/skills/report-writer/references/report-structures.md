# 汇报结构参考

> 用于 report-writer。

## 结论先行(金字塔原理)
先给结论/最重要的成果,再展开支撑。上级时间少,别让他自己找重点。

## SCQA(开场铺陈)
情境 Situation → 冲突 Complication → 问题 Question → 答案 Answer。适合项目汇报开头。

## 周报骨架
1. 一句话总结(本期最重要的进展)。
2. 本期成效:按项目分块,做了什么 → 结果(量化)→ 价值。
3. 问题与风险 + 对策。
4. 下一步计划(对齐目标)。

## 写"成果"不写"忙碌"
- 差:"处理了很多客户问题"。
- 好:"处理 37 单客诉,平均响应从 4h 降到 1h,满意度 92%"。

## 忌
流水账、只报苦劳不报功劳、无量化、报喜不报忧。
